ปัญหา: PT Medical System ทำงานบน HTTPS (GitHub Pages) แต่ CMSV6 API เป็น HTTP → Browser บล็อกด้วย Mixed Content Policy ต้องมี Proxy HTTPS → HTTP คั่น

Synology เป็น primary — ถ้า NAS down/timeout ระบบจะ fallback ไป Render → GAS อัตโนมัติ (ตั้งค่าเรียบร้อยจากใน Admin > Settings > GPS)

ข้อดีของ Synology Proxy:
  • เร็วกว่า Render (ไม่มี cold-start)
  • ไม่มีข้อจำกัด request/วัน เหมือน GAS
  • อยู่ในเครือข่ายองค์กร — ลด latency
  • ควบคุม log/audit ได้เอง

⚠️ Path mapping: DSM Reverse Proxy จะส่งทุก path ที่ตามหลัง hostname ไปยัง destination — นั่นหมายความว่า staff.supwilai.com/StandardApiAction_login.action จะไปที่ http://203.170.193.90/StandardApiAction_login.action

แต่ระบบ webapp คาดว่าเป็น staff.supwilai.com/cmsv6/StandardApiAction_login.action → ต้องใช้ Custom Header + URL rewrite ในขั้น 2.4
Step 2.3 — เพิ่ม Custom Headers (CORS + Cache)

หลังคลิก Create แล้วกดเข้าไปแก้รายการ → tab Custom Header

คลิก Create → CORS (DSM จะเติม header CORS พื้นฐานให้) → ติ๊กทุกข้อ

ถ้าอยากตั้งแบบละเอียดเอง ให้ Create → Custom Header แล้วเพิ่ม:

Header NameValue
Access-Control-Allow-Originhttps://supwilaimedical.github.io
Access-Control-Allow-MethodsGET, POST, OPTIONS
Access-Control-Allow-HeadersContent-Type, Accept
Access-Control-Max-Age86400
💡 ถ้าใช้ 2 โดเมน (supwilai + officethegood) ให้ตั้ง Access-Control-Allow-Origin: * แทน — หรือใช้ logic ใน web Station ตรวจ referer ก่อน mirror กลับ
Step 2.4 — Path Rewrite (ถ้าอยากได้ /cmsv6/ prefix)

DSM GUI ไม่รองรับ path rewrite แต่ทางเลือกคือ:

ทางเลือก A — ตั้ง Source เป็น path-level (แนะนำ)

สร้าง Reverse Proxy entry แยก (ไม่ใช่ hostname level):

ช่องค่า
Source Hostnamestaff.supwilai.com
Source Port443
Destinationhttp://203.170.193.90:80

แล้วปรับ webapp ให้ไม่ใส่ /cmsv6/ — เก็บ URL เป็น https://staff.supwilai.com เฉยๆ

ทางเลือก B — ใช้ Web Station + nginx custom config

ถ้าอยากแยก path /cmsv6/ ให้ใช้ Web Station (หรือ Container Manager + nginx):

# /volume1/web/nginx-cmsv6.conf (สร้างเพิ่มใน Web Station)
location ^~ /cmsv6/ {
    rewrite ^/cmsv6/(.*)$ /$1 break;
    proxy_pass http://203.170.193.90:80;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;

    # CORS
    add_header Access-Control-Allow-Origin  "https://supwilaimedical.github.io" always;
    add_header Access-Control-Allow-Methods "GET, POST, OPTIONS" always;
    add_header Access-Control-Allow-Headers "Content-Type, Accept" always;
    if ($request_method = OPTIONS) { return 204; }
}
⚠️ ใน webapp จะตั้ง GPS_PROXY_SYNOLOGY = https://staff.supwilai.com/cmsv6 — ระบบจะ strip http://203.170.193.90 ออกแล้ว append path ลงต่อท้าย
เช่น http://203.170.193.90/a/b?x=1 → https://staff.supwilai.com/cmsv6/a/b?x=1

3ทดสอบจาก Browser

Step 3.1 — ทดสอบ CORS preflight

เปิด browser console (F12) แล้วรัน:

fetch('https://staff.supwilai.com/cmsv6/StandardApiAction_login.action?account=supwilai&password=1234')
  .then(r => r.json())
  .then(d => console.log('✅ OK:', d))
  .catch(e => console.error('❌ Fail:', e));
  • ✅ ได้ JSON → ใช้งานได้เลย
  • ❌ CORS error → กลับไปเช็ค Step 2.3 (Custom Headers)
  • ❌ 404 Not Found → path rewrite ไม่ถูก ดู Step 2.4
  • ❌ 502 Bad Gateway → CMSV6 IP ไม่ reachable จาก NAS
  • ❌ SSL cert error → ตั้ง cert ให้ staff.supwilai.com ใน DSM > Security > Certificate

4ตั้งค่าใน PT Medical System

  1. Login เป็น Admin
  2. ไป Admin Settings → tab ตั้งค่า
  3. เลื่อนลงมาที่ GPS Tracking
  4. หัวข้อ Synology Reverse Proxy → กรอก URL เช่น https://staff.supwilai.com/cmsv6
  5. ระบบจะ auto-save + probe สถานะ → รอดูแถบ Synology (1°) เปลี่ยนเป็น ✓ ทำงานปกติ
  6. ถ้าอยากปิดใช้ Synology (กลับไปใช้ Render) กดปุ่ม × ล้างค่า

5Troubleshooting

อาการสาเหตุ + แก้
❌ CORS error ใน console Custom Header ยังไม่ตั้ง หรือ Origin ไม่ตรง → ดู Step 2.3
❌ 502 Bad Gateway NAS ต่อ CMSV6 ไม่ได้ (firewall, routing) → ทดสอบ curl -v http://203.170.193.90/ จาก SSH บน NAS
❌ SSL Handshake fail Cert หมดอายุ หรือ hostname ไม่ตรง → DSM > Security > Certificate → ใส่ Let's Encrypt ใหม่
❌ 404 ทุก endpoint Path rewrite ไม่ได้ทำ (Step 2.4) → ใช้ทางเลือก A หรือ B
⚠️ ช้าเกิน 5 วินาที Internet ขาเข้า NAS ช้า → ระบบจะ fallback ไป Render อัตโนมัติ (ไม่ต้องแก้ไข)
⚠️ Webapp ยังใช้ Render อยู่ Settings ยังไม่ save หรือยังไม่ได้ refresh page → เข้า Admin > เช็ค field Synology URL ใส่ค่าไว้ไหม แล้ว refresh GPS page

6Security แนะนำ

  • 🔒 Access Control Profile: จำกัด IP ที่เรียกได้ เฉพาะ IP ของ GitHub Pages (dynamic — อาจต้อง allow all + rely on CORS)
  • 🔒 Rate limit: ใช้ DSM > Security > Protection เปิด Auto Block + Account Protection
  • 🔒 DoS Protection: Control Panel > Security > Protection → ☑️ Enable DoS protection
  • 🔒 Log: เปิด Log Center บันทึก request ไว้ตรวจย้อนหลัง
  • 🔒 HTTPS Only: บังคับให้เข้า HTTPS เท่านั้น (redirect HTTP → HTTPS)
  • 🔒 Hide version: DSM → Advanced → ปิด Server Header ที่เปิดเผย DSM version

7Monitoring

ตรวจสถานะ Synology Proxy ได้ 2 ที่:

  • Admin > ตั้งค่า > GPS Tracking > Proxy Status — แสดง Synology (1°) พร้อม response time
  • Admin > ภาพรวม > System Health — probe อัตโนมัติทุกครั้งที่เข้า tab (แสดงสถานะเขียว/เหลือง/แดง)

ถ้า Synology down ระบบ fallback Render ได้เอง — ผู้ใช้ไม่รู้สึก
