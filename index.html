ข้อมูลระบบ
แพลตฟอร์มและสถาปัตยกรรม
  • Platform: Supabase + GitHub Pages
  • Frontend: HTML5 / JavaScript (Vanilla) / Bootstrap 5.3
  • Backend: Supabase (PostgreSQL + REST API)
  • Database: Supabase PostgreSQL
  • Architecture: Multi-Page — แต่ละระบบแยก HTML (transport, firstaid, location, gps)
ระบบที่รองรับ (4 ระบบ)
  • Patient Transport: บันทึกการนำส่งผู้ป่วย, Vitals, IV/Medication, Airway, ทีมนำส่ง, พิมพ์เอกสาร A4
  • First Aid: จัดการ Event ปฐมพยาบาล, บันทึกผู้ป่วยรายคน, Quick Dispense, Triage (AVPU), รายงานสรุป
  • PT Location: ฐานข้อมูลตำแหน่งผู้ป่วย, แผนที่ Leaflet + MarkerCluster, แชร์ลิงก์สาธารณะ
  • GPS Tracking: ติดตามรถ Real-time, รองรับหลาย Provider (CMSV6), แชร์ลิงก์ติดตาม, Proxy Fallback (Render + GAS)
ความปลอดภัยและ Session
  • Authentication: เชื่อมต่อระบบ HR Backend สำหรับยืนยันตัวตน
  • Session Token: localStorage-based session
  • Role-based Access: Admin / User — การลบเคส, จัดการ Settings, Share Token ต้องเป็น Admin
GPS Proxy Architecture
  • Primary: Render.com Node.js Proxy — HTTPS → HTTP (CMSV6), ~0.3s response
  • Fallback: Google Apps Script — สำรองเมื่อ Render cold start, timeout 5 วินาที
  • เหตุผล: GitHub Pages (HTTPS) ไม่สามารถเรียก CMSV6 API (HTTP) ตรงได้ (Mixed Content)
Libraries ที่ใช้งาน
  • Bootstrap 5.3.0 — UI Framework
  • Bootstrap Icons 1.11 — Icon Set
  • SweetAlert2 — Modal / Alert Dialogs
  • Supabase JS v2 — Backend Client
  • Leaflet 1.9.4 — Interactive Map (Lazy Loaded)
  • Leaflet MarkerCluster 1.4.1 — Map Clustering
  • Google Fonts (Sarabun) — Thai Typography
